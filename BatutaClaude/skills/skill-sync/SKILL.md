# Skill-Sync — Automatic Routing Table Generation

## Purpose

Maintain routing tables automatically. **No human should manually edit auto-generated sections.**

When a new skill is created or an existing skill's frontmatter changes, skill-sync reads ALL skill frontmatters and validates the ecosystem inventory. For scope agents with legacy AUTO-GENERATED tables, it regenerates them. For agents using native frontmatter `skills:` field, it validates that listed skills exist.

## How It Works

```
READS (sources of truth):
  BatutaClaude/skills/*/SKILL.md   → frontmatter: scope, auto_invoke, allowed-tools
  BatutaClaude/agents/*.md          → detects which agents exist

VALIDATES:
  BatutaClaude/agents/*-agent.md    → skills in frontmatter match existing SKILL.md files

WRITES (only for legacy agents with AUTO-GENERATED delimiters):
  BatutaClaude/agents/*-agent.md    → Section "### Skills" (per-scope table)

NOTE: CLAUDE.md no longer has an AUTO-GENERATED table since v8.
Skills are auto-invoked by Claude Code via their description field.
```

## Delimiter Convention

Auto-generated sections are delimited by HTML comments:

```markdown
### Available Skills
<!-- AUTO-GENERATED by skill-sync — do not edit manually -->

| Skill | Scope | Auto-invoke |
|-------|-------|-------------|
| ... |

<!-- END AUTO-GENERATED -->
```

- `sync.sh` ONLY replaces content between these delimiters
- Content before and after the delimiters is NEVER touched
- If delimiters don't exist in a target file, sync warns but does NOT create them

## Script Reference

Main script: `BatutaClaude/skills/skill-sync/assets/sync.sh`

### Usage

```bash
# Sync all scopes
bash BatutaClaude/skills/skill-sync/assets/sync.sh

# Preview changes without writing
bash BatutaClaude/skills/skill-sync/assets/sync.sh --dry-run

# Sync only one scope
bash BatutaClaude/skills/skill-sync/assets/sync.sh --scope pipeline

# Verbose output
bash BatutaClaude/skills/skill-sync/assets/sync.sh --verbose
```

### What It Does (Step by Step)

1. Find repo root (looks for `BatutaClaude/` directory)
2. Scan all `BatutaClaude/skills/*/SKILL.md` files
3. Extract YAML frontmatter: `name`, `scope`, `auto_invoke`, `allowed-tools`
4. Validate required fields — report any skills missing them
5. Group skills by scope (pipeline, infra, observability, etc.)
6. For each scope with an agent file:
   - If agent has AUTO-GENERATED delimiters (legacy): generate and replace table
   - If agent uses native frontmatter: validate skills exist
7. Report summary: skills synced, warnings, scopes without agents

## Table Formats

### In scope-agent files (e.g., pipeline-agent.md)

```markdown
| Skill | Auto-invoke | Tools |
|-------|-------------|-------|
| `sdd-init` | Starting SDD workflow, /sdd-init | Read, Edit, Write, Glob, Grep, Bash |
```

### In CLAUDE.md

```markdown
| Skill | Scope | Auto-invoke |
|-------|-------|-------------|
| `sdd-init` | pipeline | Starting SDD workflow, /sdd-init |
```

## Ordering

Tables are sorted deterministically: by scope (alphabetical), then by skill name (alphabetical). Uses `LC_ALL=C sort` to ensure consistent ordering across platforms.

## Frontmatter Requirements

Every SKILL.md MUST have these fields for skill-sync to work:

| Field | Location | Required | Example |
|-------|----------|----------|---------|
| `name` | Top level | Yes | `sdd-init` |
| `metadata.scope` | Inside metadata | Yes | `[pipeline]` |
| `metadata.auto_invoke` | Inside metadata | Yes | String or YAML list |
| `allowed-tools` | Top level | Yes | `Read, Edit, Write, Glob, Grep, Bash` |
| `platforms` | Top level | Yes | `[claude, antigravity]` or `[claude]` |

If `auto_invoke` is a YAML list, items are joined with "; " for display in the table.

### Platforms Field

The `platforms` field controls which platforms receive the skill during sync:
- `[claude, antigravity]` — Default. Skill works on both Claude Code and Antigravity.
- `[claude]` — Claude-only. Skill requires features not available in Antigravity (hooks, Agent Teams).
- `infra/sync.sh --to-antigravity` uses this field to filter which skills are copied to `BatutaAntigravity/skills/`.

## Integration Points

### With setup.sh

`setup.sh --all` runs skill-sync BEFORE copying CLAUDE.md to project root:
```
sync skills → sync agents → run skill-sync → copy CLAUDE.md
```

### With ecosystem-creator

After creating a new skill via `/create-skill`, the ecosystem-creator should suggest:
> "Skill creado. Ejecuta `/batuta:sync-skills` para actualizar las tablas de routing."

### With /batuta:sync-skills command

The slash command `BatutaClaude/commands/batuta-sync-skills.md` invokes this script directly.

## Idempotency and Safety

- Multiple runs produce identical output (deterministic)
- Only replaces content between `<!-- AUTO-GENERATED -->` delimiters
- Content outside delimiters is NEVER modified
- `--dry-run` shows what would change without writing
- Skills missing required metadata are reported but don't block sync of valid skills
